### История, импорт по URL, настройки

- **Экспорт**: после успешной генерации **одного** `.cfg` (вкладка «Один файл») в локальную историю пишется такой же JSON-снимок, как для модульного набора.
- **История**: команда **`history_clear`**, кнопка «Очистить всё…» на странице профилей; лимит хранимых записей (**`history_max_entries`**, 10–500) в **`app_settings`** с обрезкой старых записей при добавлении снимка и при сохранении лимита в **«Настройки»**.
- **Импорт по ссылке**: **`fetch_text_from_url`** (только `http`/`https`, ответ до 512 КБ, UTF-8) и карточка **«По ссылке»** на странице импорта (настольное приложение).
- **CI**: workflow **`.github/workflows/ci.yml`** — `cargo test`, `cargo check`, `npm ci`, `npm run build`.
- **Тесты**: в **`goldsr_cfg_core`** добавлена проверка непустого вывода **`buyscripts.cfg`**.

### Архитектура и backend (Фаза 1)

- **`goldsr_cfg_core`**: модульный экспорт — `exporter.rs` (`ModularFile`, `generate_modular_files`: `config/*.cfg`, бинды, `autoexec.cfg` с цепочкой `exec`); парсинг импорта — `importer.rs` (`parse_cfg_text_to_config`, бинды и CVAR, зависимость **`regex`**); у **`CfgConfig`** включена сериализация **`serde`** для обмена с Tauri; в **`generator`** вынесены общие части для экспорта (`CAT_ORDER`, категоризация, баннер консоли).
- **`src-tauri`**: локальная **SQLite** (`rusqlite`, bundled) — `AppDb` в каталоге данных приложения, таблицы **`profiles`**, **`history`**, **`app_settings`**; команды **`profile_save`**, **`profile_list`**, **`profile_load`**, **`profile_delete`**.
- **Конфиг и игра**: **`export_modular_config`**, **`parse_import_cfg`** (JSON конфигурации для редактора); **`generate_config_from_json`**, **`export_modular_from_json`** — один файл или модульный набор из JSON **`CfgConfig`**; **`get_aliases_catalog`** — список алиасов из JSON; **`detect_game_installation`** (типовые пути Steam на Windows), **`deploy_modular_files`**, **`execute_console_command_stub`** — заглушка с понятной ошибкой.
- **`aliases` (core)**: генерация **`config/aliases.cfg`** из **`data/aliases.json`** (~71 пункт: +/-, simple, cycle, toggle, kz_chain); пресеты **`minimal` / `full` / `custom`**, флаг **`include_practice`**, секции KZ/Surf по **`mode_key`**.

### Клиент

- **`ExportPage`**: вкладка **«Модульный экспорт»** — сборка набора через IPC, предпросмотр по файлам, в настольной сборке — запись всего набора в папку через диалог.
- **`src/lib/api.ts`**, **`src/types/api.ts`**: обёртки и типы под новые команды (**`ModularFile`**, профили, детект игры, деплой); **`parseImportCfg`** с Zod (**`cfgConfigParsedSchema`**); **`generateConfigFromJson`**, **`exportModularFromJson`**.
- **`MainLayout`**: при запуске в Tauri подгрузка списка профилей через **`refreshProfiles`** в store.
- **`src/lib/idbStorage.ts`**: реэкспорт **`idb-keyval`** для будущего persist (IndexedDB).
- **`ImportPage`**: в программе — разбор структуры (числа CVAR / биндов), **«Сохранить как профиль»**, ссылка на профили; **`configStore`**: черновик **`stagedProfileJson`**.
- **`ProfilesPage`** (`/profiles`): список профилей, один `.cfg`, модульный деплой в папку, просмотр, загрузка в черновик, удаление; в браузере — пояснение.
- **`DashboardPage`**: карточка «Локальные профили» (только Tauri).
- **`SettingsPage`**: **«Найти установку»** — **`detect_game_installation`** (Tauri).
- **`PreviewPage`**: приход с **`/profiles`** с текстом сгенерированного `.cfg` через `location.state`.
- **`hooks/useProfiles.ts`**: обёртка над store профилей.
- **`AliasesPage`** (`/aliases`): пресеты алиасов, практика, ручной выбор всех пунктов; **`configStore`** — **`aliasPreset`**, **`includePractice`**, **`aliasEnabled`**; экспорт и генерация на главной / режимах / пресетах передают опции в **`generate_config`** / **`export_modular_config`**.

### UX

- **`src/lib/aliasPrefs.ts`**: сохранение настроек алиасов в **`localStorage`**, гидратация и автозапись из **`MainLayout`**.
- **Ctrl+Shift+A / ⌘⇧A** — переход на **`/aliases`** (вне полей ввода); запись в **`keyboardShortcutsMeta`** и таблице F1.
- **Главная** — быстрое действие «Алиасы»; **диагностика** — строка проверки каталога алиасов (в Tauri — через IPC).
